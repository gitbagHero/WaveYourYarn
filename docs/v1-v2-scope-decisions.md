# WaveYourYarn v1.0 / v2.0 范围决策记录

> 决策日期：2026-07-21
>
> 决策状态：已确认
>
> 适用范围：v0.3.x–v2.0.x
>
> 维护规则：本文件记录影响 v1.0 产品边界、发布方式和 v2.0 高级能力的正式决策。后续修改必须同步更新 `docs/project-roadmap.md`、`docs/system-design.md` 和 README。

## D1：v1.0 定位为只读音乐数据与 AI 报告工具

决定：

- v1.0 只读取网易云账号、歌曲、liked 和歌单数据；
- v1.0 不创建、删除、覆盖、重命名或修改任何网易云远端内容；
- AI 只用于本地统计解释和报告生成，不获得网易云账号写权限；
- 报告重命名、删除和本地缓存清理属于本地数据管理，不属于远端账号写入；
- renderer、preload、IPC 和 main service 均不增加网易云写入能力。

原因：

- 产品当前定位是查看、分析和导出个人音乐数据的辅助系统；
- 完全移除 v1.0 的远端写入面，可以降低非官方接口变化、误操作、重复执行和账号风险；
- 当前代码尚无网易云写 adapter，不存在需要保留的写入兼容负担。

## D2：v0.4.x 与 v0.5.x 整体迁移到 v2.0

决定：

- 不发布原规划中的 v0.4.x AI 歌单整理系列；
- 不发布原规划中的 v0.5.x Apple Music 迁移系列；
- 两组能力统一进入 v2.0“高级歌单管理”产品线；
- v0.4.x、v0.5.x 编号保留为历史规划记录，不复用于其他功能；
- v1.0 的数据库、IPC 和 UI 不提前加入半成品歌单写入或 Apple Music 功能。

v2.0 候选范围：

- AI 生成新的推荐歌单方案；
- 原始歌单与系统生成歌单的严格隔离；
- 用户确认后向网易云发布全新的推荐歌单；
- Apple Music 授权、曲目匹配、人工复核与迁移；
- 统一的方案、匹配、发布、迁移、审计和失败恢复模型。

以下问题保留到 v2.0 立项时决定：

- 新歌单发布的唯一高层接口和内部允许能力；
- 如何证明目标歌单由当前任务新建；
- 网络结果未知时的幂等与人工复核流程；
- 部分失败后是否允许继续向系统生成歌单追加歌曲；
- 网易云发布与 Apple Music 迁移是否共享任务模型。

无论 v2.0 最终如何设计，都不得删除、覆盖或更新用户原有远端歌单。

## D3：v0.6.x 聚焦 Windows，而不是扩大到全部桌面平台

决定：

- v0.6.x 将现有 macOS 应用扩展到 Windows；
- 首个正式目标为 Windows x64；
- macOS arm64/x64 继续支持；
- Linux 不进入产品支持、打包和人工回归范围；
- 可以保留 Linux CI 执行无凭据的类型检查、测试和构建，但不据此宣称 Linux 受支持。

v0.6.x 必须覆盖：

- Windows 原生依赖构建；
- `safeStorage`、数据库、备份恢复和路径行为；
- 中文路径、长路径、导出文件名和系统对话框；
- 网页登录窗口与 session；
- Windows x64 安装包或便携包；
- macOS/Windows 源码与打包态 E2E。

## D4：签名、公证和自动更新不阻塞 v1.0

决定：

- v1.0 允许发布未签名、未公证的 macOS 和 Windows 开源安装包；
- macOS Developer ID、Apple 公证和 Windows 代码签名不作为 v1.0 发布门禁；
- v1.0 不接入自动更新通道；
- Release 必须明确说明未签名风险、安装方式和支持边界；
- Release 必须提供 SHA-256 校验值，并保留可审计的公开构建流程；
- 应用不得尝试自动关闭、修改或绕过操作系统安全机制。

说明：MIT License 与代码签名解决的问题不同。MIT 允许使用、修改和分发并包含免责声明；未签名发行仍会产生系统警告和安装摩擦。因此本决策是范围与成本取舍，不代表签名没有价值。签名、公证和安全更新可以在后续版本重新评估。

## D5：v0.7.x 是独立 UI 与用户体验版本

决定：

- v0.6.x 完成 Windows 支持后，新增 v0.7.x UI/UX 专项；
- v0.7.x 不增加新的业务领域和远端能力；
- v0.7.x 同时优化 macOS 与 Windows，完成后才进入 v1.0 发布候选阶段。

v0.7.x 范围：

- 统一视觉系统、颜色、字体、间距和组件状态；
- 优化导航、信息层级和页面布局；
- 完善首次使用、网易云登录、LLM 配置和数据披露引导；
- 统一 loading、empty、error、offline 和 partial 状态；
- 改善表格、搜索、统计、报告阅读与导出体验；
- 完善键盘导航、焦点、对比度和基础可访问性；
- 适配不同窗口尺寸、DPI 和 macOS/Windows 差异；
- 优化大型本地数据集的列表与页面响应性能；
- 建立关键页面的视觉回归与人工验收清单。

## D6：v1.0 只做稳定性收口和 Release

决定：

- v0.7.x 完成后冻结 v1.0 功能；
- 先发布至少一个 `v1.0.0-rc`，再发布 `v1.0.0`；
- v1.0 不临时加入歌单整理、远端写入、Apple Music、Linux、签名、公证或自动更新；
- v1.0 发布物为 GitHub Release 中的 macOS arm64/x64 与 Windows x64 未签名产物及校验值。

v1.0 核心门禁：

- 从 v0.2.4 数据库连续迁移到最新 schema；
- 登录、同步、离线查看、搜索、导出、统计、AI 报告、报告历史和报告导出通过回归；
- 网易云远端全程只读；
- Cookie 和 LLM API Key 不进入 renderer、日志、诊断或普通备份；
- 备份恢复、崩溃恢复和跨平台数据兼容经过验证；
- macOS/Windows 打包态 E2E 和真实账号只读人工矩阵完成；
- README、隐私说明、安装说明、已知限制和故障排查完整；
- `main`、tag、package version 和发布产物版本一致。

# 2. 本阶段目标

本阶段目标是基于当前已经同步到 SQLite 的“我喜欢的音乐”数据，实现真实导出链路。

用户应该可以：

```text
1. 打开“数据导出”页面；
2. 选择导出格式：CSV / JSON / Markdown；
3. 选择导出范围：全部喜欢歌曲 / 当前搜索筛选结果；
4. 选择排序方式；
5. 选择导出路径；
6. 点击导出；
7. 本地生成导出文件；
8. 查看导出历史；
9. 打开导出文件；
10. 打开导出文件所在目录。
```

本阶段完成后，WaveYourYarn 的 MVP 核心闭环将基本成立：

```text

同步“我喜欢的音乐”

保存收藏时间与顺序
```

# 5. 数据来源要求

本阶段导出的数据必须来自本地 SQLite 缓存，而不是重新请求网易云 API。

也就是说，导出流程应该是：

```text
读取本地 LikedSong[]
  ↓
根据导出范围筛选
  ↓
根据排序方式排序
  ↓
转换为目标格式
  ↓
写入本地文件
  ↓
保存导出记录
```

不要在导出时调用：

```text
网易云 API
api-enhanced
playlist_detail
song_detail
likelist
```

导出功能只处理本地缓存数据。

# 9. CSV 导出要求

## 9.1 CSV 字段

CSV 表头：

```text
序号,歌名,歌手,专辑,时长,收藏时间,网易云歌曲ID,封面链接,网易云原始顺序
```

字段说明：

```text
序号：当前导出列表中的序号，从 1 开始；
歌名：song.name；
歌手：artists.join(' / ')；
专辑：album；
时长：格式化为 mm:ss；
收藏时间：格式化为本地时间字符串，缺失时为空；
网易云歌曲ID：ncmSongId；
封面链接：coverUrl；
网易云原始顺序：orderIndex + 1。
```

------

## 9.2 CSV 转义

必须正确处理：

```text
逗号
双引号
换行符
中文
空字段
```

CSV 单元格转义规则：

```ts
function escapeCsvCell(value: unknown): string {
  const text = value == null ? '' : String(value)
  const escaped = text.replace(/"/g, '""')
  return /[",\n\r]/.test(escaped) ? `"${escaped}"` : escaped
}
```

------

## 9.3 UTF-8 BOM

为了让 Excel / Numbers 更好识别中文，CSV 文件建议写入 UTF-8 BOM：

```ts
const content = '\uFEFF' + csvContent
```

# 10. JSON 导出要求

JSON 应包含元信息和歌曲列表。

结构示例：

```json
{
  "schemaVersion": 1,
  "app": "WaveYourYarn",
  "source": "Netease Cloud Music",
  "playlist": "我喜欢的音乐",
  "exportedAt": "2026-07-01T00:00:00.000Z",
  "format": "json",
  "sortMode": "likedAtDesc",
  "count": 2,
  "songs": [
    {
      "index": 1,
      "ncmSongId": "123",
      "name": "歌曲 A",
      "artists": ["歌手 A"],
      "album": "专辑 A",
      "durationMs": 240000,
      "duration": "04:00",
      "likedAt": "2026-07-01 12:00:00",
      "likedAtTimestamp": 1780000000000,
      "coverUrl": "https://xxx",
      "orderIndex": 0
    }
  ]
}
```

要求：

```text
1. JSON 需要格式化缩进；
2. 不导出 rawData；
3. 不导出 Cookie；
4. 不导出任何登录凭证；
5. 缺失字段使用 null 或省略均可，但同一文件内保持一致。
```

建议不导出 `rawData`，避免导出文件过大，也避免包含不必要的接口原始数据。

# 12. 排序逻辑

导出时必须支持三种排序：

```ts
type ExportSortMode =
  | 'likedAtDesc'
  | 'likedAtAsc'
  | 'originalOrder'
```

排序规则：

```text
likedAtDesc：likedAt 从大到小；likedAt 缺失时排到后面，再按 orderIndex 排；
likedAtAsc：likedAt 从小到大；likedAt 缺失时排到后面，再按 orderIndex 排；
originalOrder：orderIndex 从小到大。
```

导出页的排序选项应该与 LikedSongsPage 保持一致。

# 13. 导出范围

本阶段支持两种范围：

```text
全部喜欢歌曲
当前搜索筛选结果
```

## 13.1 全部喜欢歌曲

直接读取本地所有 `LikedSong[]`。

## 13.2 当前搜索筛选结果

导出时可以传入 `keyword`，main process 根据 keyword 重新搜索本地数据库。

不要直接让 renderer 把整份歌曲数组传给 main process 导出。

推荐方式：

```text
renderer 传 ExportOptions：
{
  scope: 'filtered',
  keyword: '周杰伦',
  format: 'csv',
  sortMode: 'likedAtDesc'
}
  ↓
main process 根据 keyword 查询 SQLite
  ↓
main process 生成文件
```

这样可以避免 renderer 和 main 传输大量数据。

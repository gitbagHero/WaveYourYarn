# v0.3 音乐偏好人格报告调研与首版结构

> 调研日期：2026-07-21
>
> 首版数据范围：最近收藏的 100 首歌曲
>
> 定位：娱乐性的音乐偏好画像，不是心理测评或人格诊断

## 1. 外部参考

主流音乐回顾产品的共同结构不是直接计算临床人格，而是先展示可验证的听歌/偏好事实，再包装成可分享的描述：

- Spotify 2022 Listening Personality 使用熟悉/探索、忠诚/多样、经典/新鲜、主流/独特四组行为维度；
- Apple Music Replay 以歌曲、艺术家、专辑、歌单、流派、播放次数和收听时长为事实层；
- YouTube Music Recap 展示头部歌曲、艺术家、情绪、流派、专辑、时间线和娱乐性角色；
- Spotify Audio Aura 把曲目映射到情绪描述，但属于产品化娱乐表达，不是经验证的心理量表。

学术研究提供了谨慎参考：

- Rentfrow、Goldberg 与 Levitin 的 MUSIC 模型把音乐偏好概括为 Mellow、Unpretentious、Sophisticated、Intense、Contemporary 五类情感/属性维度；研究基于真实音频片段反应，不能仅凭歌名完整复现；
- Nave 等人的两项大样本研究显示音乐偏好与人格特质存在可预测关联，开放性和外向性较突出，但相关关系不等于个体诊断；
- 因此 WaveYourYarn 不输出 Big Five 分数，不把收藏歌曲解释为用户真实经历，只提供有证据、带置信度的音乐偏好反思。

主要来源：

- [Spotify Listening Personality](https://newsroom.spotify.com/2022-11-30/get-to-know-your-music-listening-personality-from-2022-wrapped/)
- [Apple Music Replay 方法](https://support.apple.com/en-ie/109356)
- [YouTube Music 2024 Recap](https://blog.youtube/news-and-events/2024-music-recap-youtube/)
- [Spotify Audio Aura](https://newsroom.spotify.com/2021-12-01/learn-more-about-the-audio-aura-in-your-spotify-2021-wrapped-with-aura-reader-mystic-michaela/)

## 2. 当前数据能可靠分析什么

| 维度                   | 当前证据                                      | 输出方式                         |
| ---------------------- | --------------------------------------------- | -------------------------------- |
| 样本范围               | 最近 100 首、可用总数、是否截断、时间覆盖率   | 固定事实                         |
| 艺术家/专辑偏好中心    | 头部艺术家、头部专辑、占比、样本歌曲          | 固定事实 + 简短解释              |
| 集中与多样             | 唯一艺术家/专辑数、头部占比、单次出现艺术家比 | 中等置信度倾向                   |
| 熟悉与探索             | 重复艺术家与新出现艺术家的结构                | 近似指标，不等同真实播放探索行为 |
| 合作倾向               | 多艺术家歌曲占比                              | 固定事实 + 低风险解释            |
| 收藏节奏               | 收藏时间跨度、时间覆盖、最近顺序              | 收藏行为，不描述实际收听频率     |
| 情绪纹理与使用场景     | 歌名、艺术家、专辑的语义线索                  | 低置信度、必须引用样本歌曲       |
| 娱乐性音乐人格与关键词 | 上述事实和低置信度解释的组合                  | 称号与反思，不输出心理分数       |
| 值得注意的反差与模式   | 头部偏好和长尾之间的结构                      | 带 evidence 的叙述               |

## 3. 当前数据不能可靠分析什么

最近收藏列表不是播放历史，当前不能声称：

- 用户最常播放、循环或跳过哪些歌；
- 每日/每月真实听歌时长以及昼夜场景；
- 歌曲真实流派、BPM、调性、能量、舞蹈性或情绪值；
- 歌曲发行时间，因此不能判断偏新歌还是经典曲目；
- 艺术家热度，因此不能判断主流或小众；
- 用户的年龄、性别、职业、关系经历、心理健康或 Big Five 分数；
- 歌名或歌词描述的事件真实发生在用户身上。

如果未来获得可靠的播放计数、流派、发行日期、热度和音频特征，可以新增 Spotify 式维度；在此之前，报告必须把这些项目放在 limitations，而不是让 LLM 猜测。

## 4. 首版报告结构

`AIReportContentV1` 固定包含：

1. 标题、副标题、样本说明；
2. 音乐口味摘要与 3–6 个关键词；
3. 一个娱乐性 listening archetype；
4. 四个维度：熟悉/探索、集中/多样、收藏节奏、情绪纹理；
5. 情绪与场景建议；
6. 可验证的显著模式；
7. 低置信度人格反思；
8. 至少三条固定限制说明。

每个主要洞察必须携带 `songIds` 或 `factKeys` evidence。报告请求启用 OpenAI 兼容协议的 JSON Mode；模型引用输入外歌曲、使用未知 fact key、对解释性章节给出 high confidence、缺少四个维度或返回非法 JSON 时，整份报告判定为失败，不保存为成功结果。

## 5. 首轮测试策略

- 数据源固定为“我喜欢的音乐”；
- 按收藏时间倒序，缺失时间时以 `orderIndex` 升序兜底；
- 上限固定 100 首，统计摘要也只针对同一批 100 首；
- 真实 provider 仅用于本地 development smoke test；
- CI 和自动化测试只调用本地 mock server；
- smoke test 只保存校验通过的报告，不保存原始 provider body；
- 任何鉴权、网络、结构或 evidence 失败都不生成报告文件。

# 2. 本阶段目标

本阶段目标是跑通 WaveYourYarn 的核心数据闭环：

```text
已登录网易云账号
  ↓
从 SecureStorageService 读取已保存 Cookie
  ↓
调用 @neteasecloudmusicapienhanced/api 获取当前用户 ID
  ↓
读取“我喜欢的音乐”歌曲 ID 列表
  ↓
批量获取歌曲详情
  ↓
标准化 Song 数据结构
  ↓
保存到 SQLite
  ↓
前端展示“我喜欢的音乐”歌曲列表
  ↓
支持搜索、刷新、同步状态展示
```

本阶段完成后，用户应该可以：

```text
1. 登录网易云；
2. 点击“同步我喜欢的音乐”；
3. 等待同步完成；
4. 在“我喜欢的音乐”页面看到歌曲列表；
5. 关闭应用后重新打开，仍能看到本地缓存的歌曲；
6. 可以再次手动刷新同步。
```

# 5. 后端主进程开发任务

## 5.1 Cookie 获取与登录校验

当前登录模块已经保存网易云 Cookie。

本阶段所有网易云数据请求都必须从 main process 读取 Cookie。

要求：

```text
1. 不允许 renderer 传入 Cookie；
2. 不允许 renderer 读取 Cookie；
3. 不允许在日志中输出 Cookie；
4. NCM 相关服务统一从 SecureStorageService 获取 Cookie；
5. 如果 Cookie 不存在或失效，返回未登录错误。
```

建议在 NCM Adapter 中封装：

```ts
async getStoredCookieOrThrow(): Promise<string>
```

或者在需要登录态的 Service 中封装：

```ts
async getNcmCookie(): Promise<string | null>
```

错误返回示例：

```ts
{
  success: false,
  message: '当前未连接网易云音乐，请先登录后再同步',
  error: 'NCM_NOT_LOGGED_IN'
}
```

## 5.2 完善 NCMSongService

文件位置：

```text
src/main/adapters/ncm/NCMSongService.ts
```

需要实现以下方法：

```ts
getLikedSongIds(userId: string, cookie: string): Promise<string[]>

getSongDetails(songIds: string[], cookie: string): Promise<NcmSongDetail[]>
```

如果当前项目中已有类似方法，请在原有结构上完善。

### 5.2.1 getLikedSongIds

职责：

```text
根据当前用户 ID 获取“我喜欢的音乐”歌曲 ID 列表。
```

实现要求：

```text
1. 调用 @neteasecloudmusicapienhanced/api 中喜欢音乐列表相关接口；
2. 参数中传入 userId；
3. 请求必须携带 Cookie；
4. 返回标准化后的歌曲 ID 数组；
5. 保留接口返回顺序；
6. 如果接口返回为空，返回空数组；
7. 如果接口失败，抛出带业务含义的错误。
```

注意：

```text
不同 API 包的实际接口名称可能不同，请先检查包的真实导出。
不要凭空猜测方法名。
可以先通过主进程临时调试方式确认接口返回结构，再封装。
```

可能相关的接口语义是：

```text
likelist / 喜欢音乐列表 / liked song ids
```

具体以 `@neteasecloudmusicapienhanced/api` 实际导出为准。

------

### 5.2.2 getSongDetails

职责：

```text
根据歌曲 ID 批量获取歌曲详情。
```

实现要求：

```text
1. 支持传入 songIds 数组；
2. 支持分批请求，避免一次请求过大；
3. 每批建议 100 到 500 首之间，具体根据 API 实际表现调整；
4. 请求必须携带 Cookie；
5. 返回标准化前的网易云歌曲详情数组；
6. 处理部分歌曲不存在或无权限的情况；
7. 单批失败时可记录失败，不要直接让整个同步流程崩溃。
```

建议分批逻辑：

```ts
const batchSize = 200
for (const batch of chunk(songIds, batchSize)) {
  // request song details
}
```

## 5.3 标准化 Song 数据结构

请创建或完善歌曲标准化函数。

建议文件：

```text
src/main/adapters/ncm/ncmMappers.ts
```

或者：

```text
src/main/adapters/ncm/mappers.ts
```

需要将网易云原始歌曲结构转为项目内部 `Song` 类型。

内部 Song 类型建议：

```ts
export interface Song {
  id: string
  ncmSongId: string
  name: string
  artists: string[]
  album?: string
  duration?: number
  coverUrl?: string
  alias?: string[]
  orderIndex?: number
  rawData?: unknown
  createdAt: string

```

字段映射要求：

```text

name：歌曲名
artists：歌手名数组
album：专辑名
duration：歌曲时长，单位建议保留 API 原始毫秒
coverUrl：专辑封面 URL
alias：歌曲别名数组
orderIndex：在“我喜欢的音乐”列表中的顺序
rawData：原始 API 数据，JSON 可序列化
```

注意：

```text
1. 不要假设所有字段都存在；
2. artists 为空时返回空数组；
3. album 不存在时允许为空；
4. coverUrl 不存在时允许为空；
5. rawData 保存前需要确保可以 JSON.stringify。
```

## 5.6 完善 SongService

文件位置：

```text
src/main/services/SongService.ts
```

这是本阶段核心业务服务。

需要实现：

```ts
syncLikedSongs(): Promise<SyncLikedSongsResult>

getLikedSongs(): Promise<Song[]>

searchLikedSongs(keyword: string): Promise<Song[]>

clearLikedSongsCache(): Promise<void>
```

### 5.6.1 syncLikedSongs 流程

必须按照以下流程实现：

```text
检查登录状态
  ↓
读取 SecureStorageService 中的 ncm_cookie
  ↓
获取当前用户信息 / userId
  ↓
调用 NCMSongService.getLikedSongIds(userId, cookie)
  ↓
得到 liked song ids
  ↓
按顺序生成 orderIndex
  ↓
分批调用 NCMSongService.getSongDetails(songIds, cookie)
  ↓
标准化 Song 数据
  ↓
写入 songs 表
  ↓
创建或更新 liked playlist
  ↓
写入 playlist_songs 关联关系
  ↓
保存最近同步时间
  ↓
返回同步结果
```

### 5.6.2 同步结果类型

请定义：

```ts
export interface SyncLikedSongsResult {
  total: number
  successCount: number
  failedCount: number
  failedSongIds: string[]
  syncedAt: string
}
```

### 5.6.3 同步错误处理

需要处理：

```text
未登录
Cookie 不存在
Cookie 已失效
用户信息获取失败
喜欢列表获取失败
喜欢列表为空
歌曲详情部分失败
数据库写入失败
网络错误
API 返回结构异常
```

如果部分歌曲详情获取失败，不要让整个同步失败。

返回示例：

```ts
{
  total: 1200,
  successCount: 1192,
  failedCount: 8,
  failedSongIds: ['xxx', 'yyy'],
  syncedAt: '2026-07-01T00:00:00.000Z'
}
```

# 10. 事务要求

同步歌曲时涉及多张表写入，必须使用事务。

推荐事务范围：

```text
保存 songs
  ↓
upsert liked playlist
  ↓
replace playlist_songs
  ↓
保存 last synced setting
```

如果事务失败：


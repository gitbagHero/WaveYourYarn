# WaveYourYarn v0.3.0 执行计划

> 基线：v0.2.5
>
> 开发分支：`codex/v0.3-llm-foundation`
>
> 目标：建立安全、可替换、可测试的 LLM 基础设施；v0.3.0 不生成正式 AI 音乐报告。

## 1. 开发原则

1. 网易云相关能力继续完全只读，LLM 层不能获得网易云写入接口；
2. 真实开发 provider 只用于人工联调，不成为内置厂商、默认模型或 Release 配置；
3. 业务层依赖统一协议接口，不依赖某个模型名称或厂商响应对象；
4. API Key 只在 main process 短暂使用，renderer 永远只能获知 `hasApiKey`；
5. CI、单元测试和 E2E 只使用本地 mock provider 与假 key；
6. v0.3.0 的退出结果是“可安全配置、预览披露并测试连接”，报告生成进入 v0.3.1。

## 2. 协议基线

首个 adapter 使用 OpenAI-compatible Chat Completions：

```text
POST {baseUrl}/chat/completions
Authorization: Bearer <api-key>
Content-Type: application/json

{
  "model": "<profile-model-id>",
  "messages": [
    { "role": "system", "content": "..." },
    { "role": "user", "content": "..." }
  ]
}
```

约束：

- `baseUrl` 可以包含厂商自定义路径和版本段，adapter 只追加 `chat/completions`；
- 模型 ID 完全来自 profile 或 development config，不写死在业务代码；
- Responses API 作为未来独立协议 adapter，不与 Chat Completions 响应解析混用；
- 首批连接测试只发送固定最小文本，不发送音乐数据；
- 正式报告请求必须等 dataset、披露确认和输出校验完成后才可启用。

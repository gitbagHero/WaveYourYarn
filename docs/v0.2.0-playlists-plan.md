# 2. 本阶段目标

v0.2.0 的目标是把 WaveYourYarn 从“我喜欢的音乐导出器”扩展为“本地音乐数据中心”的第一步。

本阶段要实现：

```text
已登录网易云账号
  ↓
读取当前用户的歌单列表
  ↓
区分创建歌单 / 收藏歌单 / 我喜欢的音乐
  ↓
将歌单元信息保存到 SQLite
  ↓
在前端展示歌单列表
  ↓
支持搜索、筛选、刷新
  ↓
支持进入歌单详情占位页
```

本阶段完成后，用户应该可以：

```text
1. 打开“歌单”页面；
2. 点击“同步歌单列表”；
3. 查看自己创建的歌单、收藏的歌单和“我喜欢的音乐”；
4. 查看歌单封面、名称、歌曲数量、创建者、类型、更新时间等基础信息；
5. 搜索歌单名称；
6. 按歌单类型筛选；
7. 点击某个歌单进入详情页占位页面；
8. 应用重启后仍能看到本地缓存的歌单列表。
```

# 6. 数据结构设计

## 6.1 Playlist 类型

请统一内部歌单类型。

建议在 main 与 renderer 的类型定义中使用：

```ts
export type PlaylistType =
  | 'liked'
  | 'created'
  | 'subscribed'
  | 'unknown'

export interface Playlist {
  id: string
  ncmPlaylistId: string
  name: string
  description?: string
  coverUrl?: string
  trackCount?: number
  ownerUserId?: string
  ownerNickname?: string
  type: PlaylistType
  subscribed?: boolean
  specialType?: number
  playCount?: number
  updateTime?: number
  createTime?: number
  rawData?: unknown
  createdAt: string
  updatedAt: string
}
```

字段说明：

```text
id：
  本地数据库 ID。

ncmPlaylistId：
  网易云歌单 ID。

name：
  歌单名称。

description：
  歌单描述。

coverUrl：
  歌单封面。

trackCount：
  歌曲数量。

ownerUserId：
  歌单创建者用户 ID。

ownerNickname：
  歌单创建者昵称。

type：
  liked：我喜欢的音乐；
  created：当前用户创建的歌单；
  subscribed：当前用户收藏的歌单；
  unknown：无法判断类型。

subscribed：
  网易云返回的是否收藏标记。

specialType：
  网易云特殊歌单类型，例如“我喜欢的音乐”可能为 specialType === 5。

playCount：
  播放次数。

updateTime：
  网易云歌单更新时间，时间戳。

createTime：
  网易云歌单创建时间，时间戳。

rawData：
  原始 API 数据，仅 main process / SQLite 内部保存，不直接用于 UI。
```

## 9.1 类型判断规则

请尽量按照以下顺序判断歌单类型：

```text
1. 如果 specialType === 5，判断为 liked；
2. 如果歌单名称包含“喜欢的音乐”，且 ownerUserId === currentUserId，可判断为 liked；
3. 如果 ownerUserId === currentUserId，判断为 created；
4. 如果 subscribed === true，判断为 subscribed；
5. 如果 ownerUserId !== currentUserId，判断为 subscribed；
6. 否则判断为 unknown。
```

注意：

```text
“我喜欢的音乐”不应该同时算作普通 created 歌单。
```

# 11. PlaylistService 开发任务

文件位置：

```text

```

请实现：

```ts

getPlaylistsByType(type: PlaylistType): Promise<Playlist[]>

getPlaylistById(id: string): Promise<Playlist | null>

clearPlaylistCache(): Promise<void>
```

## 11.1 syncUserPlaylists 流程

必须按照以下流程实现：

```text
检查登录状态
  ↓
读取 SecureStorageService 中保存的 ncm_cookie
  ↓
获取当前用户信息 / userId
  ↓
调用 NCMPlaylistService.getUserPlaylists(userId, cookie)
  ↓
对每个 raw playlist 做标准化
  ↓
识别 liked / created / subscribed / unknown 类型
  ↓
写入 playlists 表
  ↓
保存 playlists_last_synced_at
  ↓
保存 playlists_last_sync_result
  ↓
返回 SyncPlaylistsResult
```

注意：

```text
1. 不要在同步歌单列表时同步每个歌单的歌曲详情；
2. 本阶段只保存歌单元信息；
3. “我喜欢的音乐”可以与已有 liked playlist 记录 upsert 合并；
4. 不要删除 songs 表；
5. 不要破坏 v0.1.2 中的 liked playlist 和 playlist_songs 数据。
```

------

## 11.2 类型统计

同步完成后需要统计：

unknownCount

如果“我喜欢的音乐”本身也是当前用户创建的歌单，统计时建议优先归类为 `liked`，不要重复算入 `createdCount`。

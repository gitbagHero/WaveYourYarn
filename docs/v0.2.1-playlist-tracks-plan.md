# 2. 本阶段目标

v0.2.1 的目标是让用户不仅能看到“有哪些歌单”，还可以进入某个歌单，同步并查看该歌单内的歌曲列表。

本阶段要实现：

```text
用户进入 /playlists/:id
  ↓
查看歌单基础信息
  ↓
点击“同步歌单歌曲”
  ↓
调用 playlist_detail 获取该歌单 trackIds
  ↓
使用 trackIds 作为歌单内歌曲顺序来源
  ↓
批量调用 song_detail 获取歌曲详情
  ↓
保存 songs 与 playlist_songs 关联
  ↓
前端展示该歌单内歌曲列表
  ↓
支持搜索、排序、刷新本地缓存
```

完成后，用户应该可以：

```text
1. 在“歌单”页面点击任意歌单；
2. 进入歌单详情页；
3. 点击“同步歌单歌曲”；
4. 查看该歌单中的歌曲列表；
5. 查看歌曲名、歌手、专辑、时长、加入歌单时间、网易云 ID；
6. 搜索歌单内歌曲；
7. 按歌单原始顺序、加入歌单时间新到旧、旧到新排序；
8. 应用重启后仍能看到该歌单的本地缓存歌曲。
```

# 6. 数据结构设计

## 6.1 PlaylistTrack 类型

请新增用于“某个歌单中的歌曲”的展示类型。

建议：

```ts
export interface PlaylistTrack extends Song {
  playlistId: string
  orderIndex: number
  addedAt?: number
}
```

字段说明：

```text
playlistId：
  本地歌单 ID。

orderIndex：
  歌曲在该歌单 trackIds 中的顺序。

addedAt：
  歌曲加入该歌单的时间，来自 playlist.trackIds.at。
  对普通歌单来说，这更准确地表示“加入歌单时间”，不是用户喜欢时间。
```

注意：

```text
1. “我喜欢的音乐”里的 addedAt 可以理解为 likedAt；
2. 普通歌单里的 addedAt 应展示为“加入歌单时间”；
3. subscribed 歌单里的 addedAt 通常是该歌曲被加入原歌单的时间，不是用户收藏这个歌单的时间。
```

------

## 6.2 SyncPlaylistSongsResult 类型

新增：

```ts
export interface SyncPlaylistSongsResult {
  playlistId: string
  ncmPlaylistId: string
  total: number
  successCount: number
  failedCount: number
  failedSongIds: string[]
  syncedAt: string
  hasAddedAt: boolean
}
```

字段说明：

```text
total：
  playlist.trackIds 中的歌曲总数。

successCount：
  成功获取详情并写入缓存的歌曲数量。

failedCount：
  未能获取详情的歌曲数量。

failedSongIds：
  获取详情失败的网易云歌曲 ID。

hasAddedAt：
  是否至少有一首歌曲拿到了 trackIds.at。
```

## 12.1 syncPlaylistSongs 流程

必须按照以下流程实现：

```text
检查登录状态
  ↓
读取 SecureStorageService 中的 ncm_cookie
  ↓
根据本地 playlistId 查询 Playlist
  ↓
获取 ncmPlaylistId
  ↓
调用 NCMPlaylistService.getPlaylistDetail(ncmPlaylistId, cookie)
  ↓
从 playlist.trackIds 提取 ncmSongId / orderIndex / addedAt
  ↓
批量调用 NCMSongService.getSongDetails(songIds, cookie)
  ↓
建立 detailMap: ncmSongId -> detail
  ↓
按 track index 顺序合并详情
  ↓
upsert songs
  ↓
replacePlaylistSongs(playlistId, songItems)
  ↓
更新 playlists 表中的 trackCount / updateTime / rawData 等可用字段
  ↓
保存 playlist_tracks_last_synced_at:{playlistId}
  ↓
返回 SyncPlaylistSongsResult
```

------

## 12.2 关键要求

```text
1. song_detail 返回顺序不能作为最终顺序；
2. 最终顺序必须以 playlist.trackIds 为准；
3. 部分歌曲详情获取失败时，不应导致整个同步失败；
4. failedSongIds 要记录；
5. 同步普通歌单时不要破坏 liked songs 缓存；
6. 同步 liked 歌单时，应尽量复用已有 liked 逻辑，但不要引入重复或冲突。
```

为了避免本阶段破坏“我喜欢的音乐”页面，建议：

```text
1. 如果 playlist.type === 'liked'，可以允许同步并写入同一套 playlist_songs；
2. 但 LikedSongsPage 的同步逻辑仍保留原来的 SongService.syncLikedSongs；
3. PlaylistDetailPage 只是展示 liked 歌单也可以，不要替换原有 LikedSongsPage 逻辑。
```

```text
专辑：
  album

时长：
  duration 格式化为 mm:ss

加入歌单时间：
  addedAt 格式化为本地时间；
  如果缺失，显示“未知”

网易云 ID：
  ncmSongId
```

## 17.6 排序方式

支持：

```text
网易云原始顺序
加入时间新到旧
加入时间旧到新
```

对应：

```ts
originalOrder
addedAtDesc
addedAtAsc
```

排序规则：

```text
originalOrder：
  orderIndex ASC

addedAtDesc：
  addedAt DESC，缺失排后，再按 orderIndex ASC

addedAtAsc：
  addedAt ASC，缺失排后，再按 orderIndex ASC
```

# WaveYourYarn v0.1.2.1 修复任务：我喜欢的音乐顺序与收藏时间

## 背景

当前 WaveYourYarn v0.1.2 已经可以登录网易云，并通过 `likelist` 获取“我喜欢的音乐”歌曲 ID，再通过 `song_detail` 批量获取歌曲详情，保存到 SQLite 并展示。

但现在发现一个问题：

```text
当前歌曲顺序不是网易云“我喜欢的音乐”中的真实顺序，也无法显示收藏时间。
```

原因是：

```text
@neteasecloudmusicapienhanced/api 的 likelist 接口源码注释为“喜欢的歌曲(无序)”，该接口只适合获取喜欢歌曲 ID 数组，不适合作为排序依据。
```

本次任务目标是：

```text
改用“我喜欢的音乐”对应的歌单详情 playlist_detail 作为主数据源，从 playlist.trackIds 中获取歌曲顺序和 at 时间戳，并将 at 作为 likedAt / addedAt 保存和展示。
```

------

## 1. 总体修复思路

请将当前同步流程从：

```text
likelist
  ↓
song_detail
  ↓
按 likelist 返回顺序保存
```

调整为：

```text
user_playlist 找到“我喜欢的音乐”歌单 ID
  ↓
playlist_detail 获取 playlist.trackIds
  ↓
从 trackIds 中提取 id、at、index
  ↓
song_detail 批量获取歌曲详情
  ↓
用 id 将 song_detail 结果合并回 trackIds 顺序
  ↓
保存歌曲详情
  ↓
保存 playlist_songs.order_index 和 playlist_songs.added_at
  ↓
前端按 likedAt / addedAt 展示和排序
```

## 2. 修改 NCMSongService

文件：

```text
src/main/adapters/ncm/NCMSongService.ts
```

新增或修改方法：

```ts
getLikedPlaylistId(userId: string, cookie: string): Promise<string>

getLikedPlaylistTrackIndex(
  playlistId: string,
  cookie: string
): Promise<Array<{
  ncmSongId: string
  orderIndex: number
  likedAt?: number
  rawTrackId?: unknown
}>>
```

### 2.1 getLikedPlaylistId

实现方式：

1. 调用 `user_playlist` 或 api-enhanced 中对应的用户歌单接口；
2. 传入当前用户 ID 和 Cookie；
3. 从返回的歌单列表中找到“我喜欢的音乐”歌单；
4. 优先匹配：
   - 歌单归属用户是当前 userId；
   - 歌单名称为 `我喜欢的音乐`；
   - 如果返回结构中存在 `specialType`、`privacy`、`subscribed`、`creator` 等字段，可以辅助判断；
5. 如果无法可靠识别，则返回明确错误，并建议后续在设置中支持用户手动选择“我喜欢的音乐”歌单。

注意：

```text
不要硬编码某一个用户的歌单 ID。
不要假设 user_playlist 只返回 30 条，必要时分页获取。
```

------

### 2.2 getLikedPlaylistTrackIndex

实现方式：

1. 调用 `playlist_detail`；
2. 传入“我喜欢的音乐”歌单 ID；
3. 读取返回值中的 `playlist.trackIds`；
4. 对每个 trackId 提取：
   - `id` → `ncmSongId`
   - `at` → `likedAt`
   - 数组下标 → `orderIndex`
5. 如果 `at` 不存在，允许为空；
6. 保留 `rawTrackId` 方便调试；
7. 不要把完整 Cookie 写入日志。

返回结构：

```ts
export interface LikedTrackIndexItem {
  ncmSongId: string
  orderIndex: number
  likedAt?: number
  rawTrackId?: unknown
}
```

## 5. 修改 SongService.syncLikedSongs

文件：

```text
src/main/services/SongService.ts
```

将当前流程改为：

```text
读取 Cookie
  ↓
获取当前用户
  ↓
获取“我喜欢的音乐”歌单 ID
  ↓
playlist_detail 获取 trackIds
  ↓
得到 likedTrackIndexItems
  ↓
提取 songIds
  ↓
song_detail 批量获取详情
  ↓
建立 detailMap: ncmSongId -> detail
  ↓
按 likedTrackIndexItems 顺序合并详情
  ↓
保存 songs
  ↓
保存 liked playlist
  ↓
保存 playlist_songs，包括 order_index 和 added_at
  ↓
返回同步结果
```

关键要求：

```text
song_detail 返回顺序不能作为最终顺序。
最终顺序必须以 playlist.trackIds 为准。
```

合并逻辑示例：

```ts
const detailMap = new Map(details.map(detail => [String(detail.id), detail]))

const likedSongs = trackIndexItems
  .map((item) => {
    const detail = detailMap.get(item.ncmSongId)
    if (!detail) return null

    return mapNcmSongToSong(detail, {
      orderIndex: item.orderIndex,
      likedAt: item.likedAt
    })
  })
  .filter(Boolean)
```

同步结果中可以新增：

```ts
export interface SyncLikedSongsResult {
  total: number
  successCount: number
  failedCount: number
  failedSongIds: string[]
  syncedAt: string
  hasLikedAt: boolean
}
```

其中：

```text
hasLikedAt = 是否至少有一首歌拿到了 likedAt
```

## 9. 排序模式

请在前端添加一个简单排序选择：

```text
排序方式：
- 按收藏时间从新到旧
- 按收藏时间从旧到新
- 按网易云原始顺序
```

排序逻辑：

```ts
newest: likedAt desc
oldest: likedAt asc
original: orderIndex asc
```

如果 `likedAt` 不存在，则回退到 `orderIndex`。

## 11. 兼容与兜底策略

```ts

```

```text
当前未能获取网易云收藏时间，歌曲顺序可能与网易云客户端不一致。
```

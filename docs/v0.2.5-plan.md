# WaveYourYarn v0.2.5 发行与质量基础开发计划

> 计划日期：2026-07-20
>
> 基线版本：v0.2.4（commit `166ac68`）
>
> 当前状态：开发完成并发布；2026-07-21 准入审计发现远程 E2E native rebuild 门禁问题，本地修复已验证，等待远程复核
>
> 版本定位：在进入 AI 能力前，补齐桌面应用的真实运行时验证、安全边界、恢复诊断和开发发行能力。

## 1. 版本目标

v0.2.5 不扩展音乐业务范围，目标是把现有闭环从“源码可运行”提升到“安装包可验证、故障可定位、数据可恢复、外部边界可审计”。

本版本完成后，应稳定支持：

```text
开发构建 / 安装包启动
  → preload bridge 可用
  → 本地数据库安全升级
  → 网易云登录与只读同步
  → 本地查看、统计和导出
  → 数据备份 / 恢复
  → 脱敏诊断
```

核心成功条件：

1. 生产构建和开发安装包都能发现 preload 缺失或加载失败；
2. renderer 不能导航到任意远端页面，也不能请求任意系统权限；
3. 非可信页面或 frame 不能调用业务 IPC；
4. macOS 当前架构可以生成未签名开发包并完成核心只读回归；
5. 用户数据库可以备份，恢复前经过格式、版本和完整性检查；
6. 诊断资料默认脱敏，不包含 Cookie、Token、API Key 或歌曲明细；
7. Node 22 下自动化检查保持无真实账号、无真实凭据可运行。

## 2. 本版本不做

- LLM Provider、API Key 配置或真实 AI 报告；
- 网易云歌单创建、歌曲添加、删除或覆盖；
- Apple Music 授权、匹配或迁移；
- 自动更新默认开启；
- macOS 正式签名、公证和公开下载页；
- Windows 正式发行；
- 云端账号、云同步或遥测平台；
- 全量 UI 重设计。

## 3. 技术决策

### 3.1 运行时验证

- 使用 Playwright 的 Electron automation 启动生产构建；
- 首批 E2E 只覆盖应用启动、preload bridge、`app:ping` 和基础路由；
- E2E 使用临时 `userData`，不得读取或修改真实用户数据库；
- Linux CI 使用虚拟显示运行 Electron；
- 真实网易云账号仍通过发布前人工清单验证。

### 3.2 安全边界

- 主窗口只加载本地生产页面或配置的开发服务器；
- 主窗口禁止页面发起任意导航；
- 外链只允许明确列出的 HTTPS 域名；
- 网易云登录窗口只允许网易云受信域导航，不注入 preload；
- 所有 session 默认拒绝权限请求；
- 所有 IPC handler 通过同一注册入口校验主 frame 和 renderer URL；
- renderer 使用 CSP 限制脚本、frame、object 和网络连接。

### 3.3 开发发行

- 使用 electron-builder；
- 固定 `appId`、`productName`、产物目录和命名规则；
- 原生 `.node` 模块不打入 asar；
- 第一阶段先生成当前架构的 `dir` 开发包；
- arm64/x64 的 DMG/ZIP 使用各自原生 GitHub runner 构建；签名和公证留待后续版本；
- 自动更新依赖虽已存在，但本版本不连接发布源。

### 3.4 恢复与诊断

- 备份是 SQLite 数据库的一致性快照，不直接复制正在写入的文件；
- 恢复前自动备份当前数据库；
- 恢复文件必须通过 SQLite header、schema version 和 integrity check；
- 诊断包只包含版本、平台、schema、路径可用性、脱敏日志和数量摘要；
- 用户需要主动选择保存位置，应用不自动上传诊断资料。

## 5. 自动化验证矩阵

| 层级            | 必须覆盖                                | 是否需要凭据     |
| --------------- | --------------------------------------- | ---------------- |
| Unit            | URL 策略、脱敏、备份 manifest、参数解析 | 否               |
| Repository      | migration、备份恢复、事务和完整性       | 否               |
| IPC integration | sender、参数、错误 envelope             | 否               |
| Electron E2E    | 启动、preload、ping、路由、CSP          | 否               |
| Package smoke   | 解包产物启动、preload、数据库初始化     | 否               |
| Manual          | 网页登录、Cookie 登录、同步、导出       | 是，本地人工执行 |

## 6. 发布阻断条件

以下任一情况存在时不得发布：

- 生产构建或安装包出现 `Electron preload API is unavailable`；
- 主窗口可以导航到未授权远端页面；
- 未授权页面、子 frame 或登录窗口可以调用业务 IPC；
- 任意权限请求被默认批准；
- 打包遗漏 migration、preload 或原生 SQLite 模块；
- 备份恢复可能覆盖当前数据库且没有恢复前快照；
- 日志、诊断包或报错包含完整 Cookie、Token 或 API Key；
- v0.2.4 数据库无法升级或完整性检查失败；
- 自动化检查失败；
- 真实账号只读闭环尚未完成人工验证。

## 7. 依赖与风险

| 风险                                  | 影响                | 控制措施                                       |
| ------------------------------------- | ------------------- | ---------------------------------------------- |
| Electron 与 better-sqlite3 ABI 不一致 | 安装包启动失败      | 每个目标架构重建原生依赖并做 package smoke     |
| CSP 在开发/生产协议下行为不同         | 页面空白或 HMR 失效 | 分别验证开发服务器和本地文件构建               |
| 网易云登录跨域跳转变化                | 登录窗口被策略阻断  | URL 策略集中维护，记录被拒绝域名且不记录 query |
| E2E 污染真实 userData                 | 用户数据风险        | 强制临时路径并在测试后清理                     |
| x64 交叉构建原生模块失败              | 无法一次生成双架构  | 优先各架构原生 runner，通用包延后评估          |
| 签名凭据缺失                          | 无法公开分发        | 开发包与正式签名流程分离                       |

## 9. 完成状态

v0.2.5 的代码与无凭据自动化按以下顺序完成：

```text
安全策略与 Electron E2E
  → 双架构开发打包
  → 备份、恢复与回滚
  → 日志、脱敏与诊断
  → 结构治理与发布资料
```

macOS arm64 包已在本机原生执行 E2E，x64 包已通过 Rosetta 执行相同 E2E。真实网易云账号的完整人工矩阵不能由无凭据自动化替代，继续使用 `docs/v0.2.5-manual-regression.md` 记录后续兼容性验证。

进入 v0.3.x 前的补充审计、远程 workflow 事实和门禁修复记录见 `docs/v0.2.5-readiness-audit.md`。

# WaveYourYarn v0.3.x 产品与技术决策记录

> 决策日期：2026-07-21
>
> 决策状态：已确认
>
> 适用范围：v0.3.0–v0.3.2
>
> 维护规则：本文件记录已经由项目所有者确认、会影响数据模型、网络边界或用户交互的决策。后续修改必须同时更新 `docs/v0.3.x-plan.md` 与 `docs/project-roadmap.md`。
>
> 跨版本说明：v1.0 只读边界、v0.4.x/v0.5.x 迁移到 v2.0、Windows、UI/UX 和未签名发行决策记录在 `docs/v1-v2-scope-decisions.md`。

## D1：多 LLM profile 与统一配置界面

决定：

- 支持多个 LLM profile，但任一时刻只有一个活动 profile；
- 提供统一的“添加 API 配置”与模型管理界面；
- 用户可自主配置名称、协议类型、Base URL、API Key、模型标识、超时、输出模式、语言和输入数量；
- API Key 只进入 `SecureStorageService`，profile 表只保存 secret reference 和公开配置；
- 业务层依赖统一 `LLMProvider` 接口，不绑定单一厂商；
- provider 使用可注册的协议 adapter。v0.3.0 必须交付 OpenAI-compatible adapter，同时保证后续增加 Anthropic-compatible、Gemini-compatible 等主流协议时不需要重做数据库、IPC 和设置页面；
- 不提供让用户从 renderer 注入任意请求代码、任意 header 或自定义脚本的能力。

原因：

- 国内外不同服务通常使用不同 Base URL 和模型名；
- 用户可能同时使用云端服务与本地兼容服务；
- 多 profile 可以避免反复覆盖 key 和模型配置；
- 协议 adapter 隔离第三方差异，避免 provider 响应结构进入业务层。

## D2：不提供应用内代理配置

决定：

- v0.3.x 不增加 system/direct/fixed、HTTP、SOCKS 或 PAC 代理设置；
- 不把 `127.0.0.1:10090` 或其他代理地址写入默认设置；
- LLM 与网易云请求继续使用运行环境的正常网络能力；
- v0.3.x 以国内可直接访问的 API 和用户自主配置的 Base URL 为主要场景；
- URL 安全策略、HTTPS/localhost 限制、超时、取消、重定向复核和日志脱敏仍然必须实现；
- 如果未来出现明确的产品需求，再单独设计代理能力，不提前保留半成品 UI 或公开设置字段。

原因：

- 当前目标 API 不依赖代理；
- 应用内代理会扩大网络、凭据、诊断和跨协议测试范围；
- 代理不是 AI 报告核心能力，不应阻塞 v0.3.x。

## D3：数据披露确认可记住且可撤销

决定：

- 默认在真实报告生成前显示数据披露确认弹窗；
- 弹窗提供“记住此选择”；
- 设置页提供可随时调整的 AI 数据披露确认模式，并可清除已经记住的授权；
- 记住的授权按 profile、目标 origin、数据来源和字段集合绑定；
- profile 目标域名、协议、数据来源、字段集合或输入上限扩大时，旧授权失效并重新弹窗；
- 连接测试只发送最小固定文本，不发送音乐数据，因此不复用报告生成授权；
- 重试或重新生成只有在当前请求仍匹配有效授权时才可跳过弹窗，否则必须重新确认；
- 即使跳过弹窗，生成页面仍展示本次将使用的 provider、模型、来源和歌曲数量摘要。

建议数据模型：

- 公开设置 `ai_disclosure_confirmation_mode`：`always` 或 `allow_remembered`；
- schema 7 增加 `ai_disclosure_consents`，保存 profile ID、target origin、source、fields hash、输入上限和更新时间；
- 表中不保存歌曲明细、prompt、API Key 或 provider 原始数据。

## D4：PDF 不是 v0.3.2 硬交付项

决定：

- Markdown 与 HTML 是 v0.3.2 的稳定交付承诺；
- PDF/打印只进行技术与布局验证；
- 只有中文字体、分页、长段落、证据块和跨平台输出达到发布质量时才加入 v0.3.2；
- 若未达标则延后，不以低质量 PDF 阻塞 Markdown/HTML 发布。

## 决策对版本范围的影响

| 版本   | 影响                                                                      |
| ------ | ------------------------------------------------------------------------- |
| v0.3.0 | 多 profile、统一配置 UI、协议 registry、OpenAI-compatible、可记住披露授权 |
| v0.3.1 | 报告生成必须校验当前披露授权；重试和重新生成遵循 D3                       |
| v0.3.2 | Markdown/HTML 必须完成；PDF 通过质量门槛才进入稳定范围                    |

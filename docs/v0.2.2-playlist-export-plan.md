# 2. 本阶段目标

v0.2.2 的目标是将当前导出系统从：

```text
只支持导出“我喜欢的音乐”
```

扩展为：

```text
支持导出任意已同步歌单
```

本阶段完成后，用户应该可以：

```text
1. 在歌单详情页点击“导出当前歌单”；
2. 跳转到导出页，并自动选择当前歌单作为导出来源；
3. 在导出页选择导出来源：
   - 我喜欢的音乐；
   - 指定歌单；
4. 选择指定歌单；
5. 选择导出格式：
   - CSV；
   - JSON；
   - Markdown；
6. 选择导出范围：
   - 全部歌曲；
   - 关键词筛选结果；
7. 选择排序方式：
   - 原始顺序；
   - 加入时间新到旧；
   - 加入时间旧到新；
8. 导出指定歌单歌曲；
9. 在导出历史中看到导出来源；
10. 打开导出文件；
11. 打开导出文件所在目录。
```

本阶段要重点为后续扩展打好基础。导出系统不应该继续只围绕 `LikedSong[]` 设计，而应该抽象为可扩展的导出来源。

# 4. 架构边界要求

请继续遵守当前架构边界：

```text
renderer：
  只负责 UI、状态展示、用户交互；
  不直接访问 Cookie；
  不直接访问 SQLite；
  不直接访问 fs / path / shell / dialog；
  不直接调用 @neteasecloudmusicapienhanced/api。

preload：
  只暴露安全 API；
  不暴露 Node.js 原生能力；
  不暴露 Cookie；
  不暴露数据库路径。

main：
  负责读取 SQLite；
  负责文件写入；
  负责 Electron dialog；
  负责 Electron shell；
  负责导出业务流程。

services：
  负责导出业务编排。

repositories：
  负责数据库读取和导出历史记录。

renderer 不要把完整歌曲数组传给 main process 导出。
导出数据应由 main process 从本地 SQLite 查询。
```

# 5. 本阶段核心改造方向

当前 `ExportService` 很可能主要围绕：

```ts
exportLikedSongs(options)
```

本阶段需要将导出系统扩展为支持多种来源。

建议引入统一导出来源类型：

```ts
export type ExportSourceType = 'liked' | 'playlist'

export interface ExportSource {
  type: ExportSourceType
  playlistId?: string
}
```

或者：

```ts
export type ExportSource =
  | { type: 'liked' }
  | { type: 'playlist'; playlistId: string }
```

推荐使用联合类型，类型更安全。

导出主入口建议扩展为：

```ts
exportSongs(options: ExportOptions): Promise<ExportResult>
```

其中 `ExportOptions` 包含：

```ts
export interface ExportOptions {
  source: ExportSource
  format: ExportFormat
  scope: ExportScope
  keyword?: string
  sortMode: ExportSortMode
  filePath?: string
}
```

保留旧方法也可以，但建议让旧方法内部调用新方法：

```ts
exportLikedSongs(options) {
  return exportSongs({
    ...options,
    source: { type: 'liked' }
  })
}

exportPlaylistSongs(playlistId, options) {
  return exportSongs({
    ...options,
    source: { type: 'playlist', playlistId }
  })
}
```

```ts
  | { type: 'liked' }

```

说明：

```text

playlist：

```

------

## 6.4 导出排序方式

当前“我喜欢的音乐”使用：

```text
likedAtDesc
likedAtAsc
originalOrder
```

指定歌单使用：

```text
addedAtDesc
addedAtAsc
originalOrder
```

为了统一导出系统，建议定义为：

```ts
export type ExportSortMode =
  | 'timeDesc'
  | 'timeAsc'
  | 'originalOrder'
```

说明：

```text
timeDesc：
  对 liked 来源表示“收藏时间新到旧”；
  对 playlist 来源表示“加入歌单时间新到旧”。

timeAsc：
  对 liked 来源表示“收藏时间旧到新”；
  对 playlist 来源表示“加入歌单时间旧到新”。

originalOrder：
  网易云原始顺序。
```

如果当前代码已经使用 `likedAtDesc / likedAtAsc / originalOrder`，可以保留兼容：

```ts
export type ExportSortMode =
  | 'timeDesc'
  | 'timeAsc'
  | 'originalOrder'
  | 'likedAtDesc'
  | 'likedAtAsc'
  | 'addedAtDesc'
  | 'addedAtAsc'
```

但推荐在 v0.2.2 内部统一转换为：

```text
timeDesc
timeAsc
originalOrder
```

这样后续导出多个来源会更清晰。

## 6.5 通用导出歌曲行

当前存在 `LikedSong` 和 `PlaylistTrack`：

```ts
export interface LikedSong extends Song {
  orderIndex: number
  likedAt?: number
}

export interface PlaylistTrack extends Song {
  playlistId: string
  orderIndex: number
  addedAt?: number
}
```

导出时建议转换为统一的中间行：

```ts
export interface ExportSongRow {
  index: number
  name: string
  artists: string
  album?: string
  duration?: string
  durationMs?: number
  timeLabel?: string
  timeTimestamp?: number
  timeColumnName: string
  ncmSongId: string
  coverUrl?: string
  orderIndex: number
}
```

说明：

```text
timeLabel：
  格式化后的时间字符串。

timeTimestamp：
  原始时间戳。

timeColumnName：
  对 liked 来源为“收藏时间”；
  对 playlist 来源为“加入歌单时间”。
```

也可以更简单地定义：

```ts
export interface ExportSongRow {
  index: number
  name: string
  artists: string
  album: string
  duration: string
  durationMs?: number
  time?: string
  timeTimestamp?: number
  ncmSongId: string
  coverUrl?: string
  orderIndex: number
}
```

然后在生成 CSV / Markdown 表头时根据来源决定时间列名称。

## 9.3 根据导出来源读取数据

请新增内部方法：

```ts
private getSongsForExport(options: ExportOptions): Promise<{
  sourceType: ExportSourceType
  sourceId?: string
  sourceName: string
  songs: Array<LikedSong | PlaylistTrack>
  timeColumnName: string
}>
```

逻辑：

```text
source.type === liked：
  从 SongService / PlaylistRepository 读取 LikedSong[]；
```

要求：

```text

2. 导出时不要请求网易云 API；
3. 如果 playlistId 不存在，返回明确错误；
4. 如果该歌单没有同步歌曲，提示先同步该歌单歌曲；
5. 如果 liked 没有同步歌曲，提示先同步“我喜欢的音乐”。
```

## 9.5 排序逻辑

请实现统一排序函数：

```ts
sortExportSongs(
  songs: Array<LikedSong | PlaylistTrack>,
  sourceType: ExportSourceType,
  sortMode: ExportSortMode
): Array<LikedSong | PlaylistTrack>
```

排序规则：

```text
originalOrder：
  orderIndex ASC。

timeDesc：
  liked 来源使用 likedAt DESC；
  playlist 来源使用 addedAt DESC；
  时间缺失排到后面；
  时间相同或缺失时按 orderIndex ASC。

timeAsc：
  liked 来源使用 likedAt ASC；
  playlist 来源使用 addedAt ASC；
  时间缺失排到后面；
  时间相同或缺失时按 orderIndex ASC。
```

兼容旧 sortMode：

```text
likedAtDesc -> timeDesc
likedAtAsc -> timeAsc
addedAtDesc -> timeDesc
addedAtAsc -> timeAsc
```

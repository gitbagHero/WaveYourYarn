## 1. 当前项目状态

WaveYourYarn 当前已经完成：


当前项目已经具备：

```text

```

现在进入下一阶段：

```text
v0.2.3 基础音乐数据统计与 AI 分析数据底座
```

------

# 2. 本阶段目标

v0.2.3 的目标不是做复杂可视化，而是先把本地音乐数据统计能力和后续 AI 报告需要的数据摘要能力搭好。

本阶段完成后，用户应该可以：

```text
1. 打开“数据统计”页面；
2. 选择统计范围：
   - 我喜欢的音乐；
   - 指定歌单；
   - 全部已同步歌曲；
3. 查看基础概览：
   - 歌曲总数；
   - 歌手数量；
   - 专辑数量；
   - 歌单数量；
   - 最早收藏 / 加入时间；
   - 最近收藏 / 加入时间；
4. 查看 Top 歌手；
5. 查看 Top 专辑；
6. 查看按年份 / 月份分布的收藏或加入趋势；
7. 查看最近收藏 / 加入的歌曲；
8. 查看最早收藏 / 加入的歌曲；
9. 查看歌单数据概览；
10. 为后续 v0.3 AI 报告提供结构化统计摘要接口。
```

本阶段的重点是：

```text
构建可复用的数据统计服务，而不是只做一个前端页面。
```

后续 v0.3 AI 报告应能直接复用本阶段生成的统计摘要。

# 4. 架构边界要求

继续遵守当前架构边界：

```text
renderer：
  只负责 UI、状态展示、用户交互；
  不直接访问 SQLite；
  不直接访问 Cookie；
  不直接调用网易云 API；
  不直接读取本地文件系统。

preload：
  只暴露安全 API；
  不暴露 Node.js 原生能力；
  不暴露 SQLite 路径；
  不暴露 Cookie。

main：
  负责从 SQLite 查询本地数据；
  负责生成统计结果；
  负责缓存或读取统计摘要；
  不在统计阶段请求网易云 API。

services：
  负责统计业务编排。

repositories：
  负责数据库查询。

renderer 不要把完整歌曲数组传给 main process 做统计。
统计数据应由 main process 从 SQLite 查询和计算。
```

# 6. 统计来源设计

## 6.1 StatisticsSource

请定义统一统计来源类型：

```ts
export type StatisticsSource =
  | { type: 'liked' }
  | { type: 'playlist'; playlistId: string }
  | { type: 'all' }
```

说明：

```text
liked：
  只统计“我喜欢的音乐”。

playlist：
  只统计指定歌单。

all：
  统计所有已同步歌曲去重后的合集。
```

注意：

```text
all 来源需要对歌曲去重。
同一首歌可能同时出现在多个歌单中。
去重建议以 ncmSongId 为准。
```

------

## 6.2 StatisticsTimeField

不同来源的时间语义不同：

```ts
export type StatisticsTimeField =
  | 'likedAt'
  | 'addedAt'
  | 'mixed'
  | 'none'
```

说明：

```text
likedAt：
  “我喜欢的音乐”的收藏时间。

addedAt：
  普通歌单中的加入歌单时间。

mixed：
  all 来源中可能混合 likedAt 和 addedAt。

none：
  当前数据缺少时间字段。
```

## 7.3 MusicStatsOverview

```ts
export interface MusicStatsOverview {
  songCount: number
  uniqueSongCount: number
  artistCount: number
  albumCount: number
  playlistCount?: number
  earliestTime?: number
  latestTime?: number
  timeCoverageLabel?: string
}
```

说明：

```text
songCount：
  当前统计来源中的歌曲条目数。
  对 all 来源，如果未去重前有重复，可表示总条目数。

uniqueSongCount：
  按 ncmSongId 去重后的歌曲数。

artistCount：
  不同歌手数量。

albumCount：
  不同专辑数量。

playlistCount：
  all 来源下参与统计的歌单数量。

earliestTime：
  最早 likedAt / addedAt。

latestTime：
  最近 likedAt / addedAt。

timeCoverageLabel：
  可读时间范围，例如：2020-03 至 2026-07。
```

如果不想新增 Repository，也可以在 SongRepository / PlaylistRepository 中补充统计查询方法，但推荐新增独立 StatisticsRepository，便于后续 AI 分析复用。

## 8.1 需要实现的方法

```ts
getLikedSongsForStats(): LikedSong[]

getPlaylistTracksForStats(playlistId: string): PlaylistTrack[]

getAllSyncedSongsForStats(): Array<LikedSong | PlaylistTrack>

getPlaylistStatsOverview(): PlaylistStatsOverview
```

------

## 8.2 查询要求

```text
1. liked 来源：
   读取“我喜欢的音乐”对应的 LikedSong[]；
   时间字段使用 likedAt。

2. playlist 来源：
   根据 playlistId 读取 PlaylistTrack[]；
   时间字段使用 addedAt。

3. all 来源：
   读取所有已同步歌单歌曲 + 我喜欢的音乐；
   按 ncmSongId 去重；
   如果同一首歌存在多个时间字段，可保留最近的时间或保留 firstSeen / latestSeen，MVP 阶段建议保留最近时间。
```

------

## 8.3 去重规则

all 来源需要去重：

```text
key = ncmSongId
```

如果同一首歌出现多次：

```text
1. 保留歌曲基础信息；
2. time 字段优先选择最大时间戳；
3. orderIndex 可以保留第一次出现的；
4. 可在后续版本增加 appearInPlaylists 信息，本阶段不强制。
```

------

## 9.2 getMusicStatsSummary 流程

```text
解析统计来源
  ↓
从 StatisticsRepository 读取本地数据
  ↓
根据来源确定 timeField
  ↓
去重
  ↓
计算 overview
  ↓
计算 topArtists
  ↓
计算 topAlbums
  ↓
计算 timeDistribution
  ↓
计算 recentSongs
  ↓
计算 earliestSongs
  ↓
如果需要，计算 playlistOverview
  ↓
返回 MusicStatsSummary
```

------

## 9.3 Top Artists 统计规则

```text
1. 遍历每首歌的 artists；
2. 每个歌手计数 +1；
3. 如果一首歌有多个歌手，每个歌手都计数；
4. 按 count DESC 排序；
5. 默认返回 Top 20；
6. percentage 基于 uniqueSongCount；
7. sampleSongs 最多保留 3 首。
```

------

## 9.4 Top Albums 统计规则

```text
1. 按 album 名称统计；
2. album 为空时归入“未知专辑”或跳过，建议归入“未知专辑”；
3. 同名专辑可能来自不同歌手，MVP 阶段可以先按 album 名称合并；
4. 按 count DESC 排序；
5. 默认返回 Top 20。
```

------

## 9.5 时间分布统计规则

时间字段根据来源决定：

```text
liked:
  likedAt

playlist:
  addedAt

all:
  mixed time
```

要求：

```text
1. 过滤掉没有时间戳的歌曲；
2. byYear 格式为 YYYY；
3. byMonth 格式为 YYYY-MM；
4. 按时间升序返回；
5. 如果没有任何时间数据，返回空数组。
```

------

## 9.6 recentSongs / earliestSongs

```text
recentSongs：
  按时间 DESC，取前 20。

earliestSongs：
  按时间 ASC，取前 20。

没有时间字段的歌曲不参与 recent / earliest。
```

------

## 9.7 getAvailableStatisticsSources

返回可用于统计的来源列表。

应包括：

```text
1. 我喜欢的音乐；
2. 已同步过歌曲的歌单；
3. 全部已同步歌曲。
```

类型：

```ts
export interface StatisticsSourceInfo {
  type: 'liked' | 'playlist' | 'all'
  id?: string
  name: string
  timeField: StatisticsTimeField
}
```

要求：

```text
1. 如果“我喜欢的音乐”没有缓存歌曲，也可以返回，但前端需提示未同步；
2. 指定歌单建议只返回已经同步过歌曲的歌单；
3. all 来源如果没有任何歌曲，也可以返回，但统计结果为空。
```

```text

  最早 100 首；
  中间均匀采样 100 首。
```

如果实现采样复杂，可以先返回按时间排序后的前 300 首，并加 TODO。

新增或完善类型文件，例如：

```text
src/shared/types/statistics.ts
```

定义：

```ts
export type StatisticsSource =
  | { type: 'liked' }
  | { type: 'playlist'; playlistId: string }
  | { type: 'all' }

export type StatisticsTimeField =
  | 'likedAt'
  | 'addedAt'
  | 'mixed'
  | 'none'

export interface StatisticsSourceInfo {
  type: 'liked' | 'playlist' | 'all'
  id?: string
  name: string
  timeField: StatisticsTimeField
}

export interface MusicStatsOverview {
  songCount: number
  uniqueSongCount: number
  artistCount: number
  albumCount: number
  playlistCount?: number
  earliestTime?: number
  latestTime?: number
  timeCoverageLabel?: string
}

export interface ArtistStat {
  name: string
  count: number
  percentage: number
  sampleSongs: Array<{
    ncmSongId: string
    name: string
  }>
}

export interface AlbumStat {
  name: string
  artistNames: string[]
  count: number
  percentage: number
  sampleSongs: Array<{
    ncmSongId: string
    name: string
  }>
}

export interface TimeDistribution {
  byYear: Array<{
    year: string
    count: number
  }>
  byMonth: Array<{
    month: string
    count: number
  }>
}

export interface SongTimeStat {
  ncmSongId: string
  name: string
  artists: string[]
  album?: string
  time?: number
  timeLabel?: string
  orderIndex?: number
}

export interface PlaylistStatsOverview {
  totalPlaylists: number
  likedCount: number
  createdCount: number
  subscribedCount: number
  unknownCount: number
  totalTracksInPlaylists?: number
  syncedPlaylistCount?: number
}

export interface MusicStatsSummary {
  source: StatisticsSourceInfo
  overview: MusicStatsOverview
  topArtists: ArtistStat[]
  topAlbums: AlbumStat[]
  timeDistribution: TimeDistribution
  recentSongs: SongTimeStat[]
  earliestSongs: SongTimeStat[]
  playlistOverview?: PlaylistStatsOverview
  generatedAt: string
}
```

同时定义 AI 预留数据集：

```ts
export interface CompactSongForAnalysis {
  name: string
  artists: string[]
  album?: string
  time?: number
  orderIndex?: number
}

export interface MusicAnalysisDataset {
  source: StatisticsSourceInfo
  summary: MusicStatsSummary
  compactSongs: CompactSongForAnalysis[]
  generatedAt: string
}
```

# WaveYourYarn v0.2.5 完成度与 v0.3.x 准入审计

> 审计日期：2026-07-21
>
> 审计基线：`main` / `v0.2.5` / commit `b31c321`
>
> 审计目的：确认 v0.2.5 的代码、发布物和质量基础是否足以支撑 v0.3.x 开发，并明确仍需持续执行的发布兼容性工作。

## 1. 结论

v0.2.5 计划中的产品与工程能力已经实现，当前代码具备进入 v0.3.x 开发的架构基础。审计同时发现并修复了一项会导致全新安装环境中 Electron E2E 超时的工程门禁问题：`better-sqlite3` 在运行源码 Electron E2E 前没有针对 Electron runtime 重建。修复已进入 `main`，新的 Linux CI 和 macOS arm64/x64 workflow 均通过。

当前状态分为两层：

| 门禁                | 结论     | 说明                                                       |
| ------------------- | -------- | ---------------------------------------------------------- |
| v0.3.x 本地开发准入 | 通过     | Node 22 完整检查、源码 E2E、打包态 E2E、干净安装模拟均通过 |
| `main` 远程准入     | 通过     | Linux CI、macOS arm64 与 macOS x64 原生 runner 均通过      |
| v0.3.x 稳定版本发布 | 尚未通过 | v0.2.5 真实账号与 macOS 双架构人工矩阵仍需留下执行记录     |

因此，v0.2.5 的开发与自动化准入闭环已经完成，可以开始 v0.3.x 功能开发。人工真实账号矩阵不阻塞 v0.3.0 基础设施开发，但仍阻塞下一个稳定版本发布。

## 3. 自动化结果

使用 Node.js `22.22.3`、pnpm `11.7.0` 执行：

| 验证                             | 结果                                                                         |
| -------------------------------- | ---------------------------------------------------------------------------- |
| `pnpm run check`                 | 20 个测试文件、51 个测试通过；typecheck、lint、生产构建和 preload guard 通过 |
| `pnpm run test:e2e:run`          | 1 个源码 Electron E2E 通过                                                   |
| `pnpm run test:e2e:packaged`     | 1 个已打包应用 E2E 通过                                                      |
| 无 `node_modules` 的离线干净安装 | 安装、完整检查、原生模块重建和源码 E2E 通过                                  |

E2E 根因复现：

```text
干净安装
  → 直接 playwright test
  → 与 GitHub Actions 一致：测试 30 秒超时、worker teardown 30 秒超时
  → pnpm run rebuild:native
  → 相同 E2E 2.1 秒通过
```

修复策略：

1. `test:e2e:run` 自身先执行 `rebuild:native`，不再依赖调用方记住 Electron ABI 重建步骤；
2. `test:e2e` 统一复用 `test:e2e:run`；
3. `package:mac` 在双架构交叉打包成功或失败后都恢复当前主机架构的 native module，避免下一次本地 Electron 启动被 x64/arm64 产物污染。

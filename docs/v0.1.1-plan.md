# 3. v0.1.1 功能任务拆解

## 3.1 接入 @neteasecloudmusicapienhanced/api

### 任务目标

将 `@neteasecloudmusicapienhanced/api` 正式接入 Electron 主进程。

### 需要完成

- 确认当前项目已经安装 `@neteasecloudmusicapienhanced/api`；
- 如果未安装，则使用 pnpm 安装；
- 在 `src/main/adapters/ncm` 中编写真实调用逻辑；
- 不允许在 renderer 中直接调用该包；
- 所有网易云相关调用必须经过 main process；
- 所有网易云相关调用必须通过 NCM Adapter 封装。

### 注意事项

由于第三方 API 包的实际导出形式可能会变化，开发时需要先检查：

```text
包的实际导出对象
二维码登录相关接口名称
登录状态接口名称
用户信息接口名称
cookie 返回方式
请求参数格式
返回值格式
```

不要凭空猜测方法名。
如果包内方法名与预期不同，请以实际包导出为准，并在代码中用注释说明映射关系。

## 3.4 登录态存储

### 目标

登录成功后，需要保存必要的登录态，使用户关闭应用后再次打开时仍可识别登录状态。

### 建议实现

优先使用 Electron 的 `safeStorage` 对敏感信息进行加密保存。

推荐新增：

```text
src/main/services/SecureStorageService.ts
```

提供方法：

```ts
setSecret(key: string, value: string): Promise<void>
getSecret(key: string): Promise<string | null>
removeSecret(key: string): Promise<void>
```

底层可以结合：

```text
electron.safeStorage
SQLite app_settings 表
```

### 存储内容

至少需要保存：

```text
ncm_cookie
ncm_user_id
ncm_nickname
ncm_avatar_url
```

其中：

- cookie 必须作为敏感信息处理；
- 日志中不能输出 cookie；
- IPC 返回值中不能直接返回 cookie；
- renderer 不应该直接看到 cookie。

### 如果 safeStorage 暂时不可用

如果当前环境下 `safeStorage` 不可用，可以先退化为本地存储，但必须：

- 统一封装在 `SecureStorageService`；
- 在代码中标注 TODO；
- 日志中依然不能输出 cookie；
- 后续可以无痛替换为加密存储。

## 3.5 AuthService 业务层

文件位置：

```text
src/main/services/AuthService.ts
```

### 业务职责

`AuthService` 负责组合 NCMAuthService、NCMUserService、SecureStorageService 和 UserRepository。

### 需要实现的方法

```ts
getLoginQr(): Promise<{
  key: string
  qrImageBase64?: string
  qrUrl?: string
}>

checkQrStatus(key: string): Promise<{
  status: 'waiting' | 'scanned' | 'authorized' | 'expired' | 'failed'
  message?: string
  user?: UserProfile
}>

getLoginStatus(): Promise<{
  isLoggedIn: boolean
  user?: UserProfile
}>

getCurrentUser(): Promise<UserProfile | null>

logout(): Promise<void>
```

### 登录成功后的处理流程

```text
二维码状态为 authorized
  ↓
从返回值中获取 cookie
  ↓
保存 cookie 到 SecureStorageService
  ↓
调用 NCMUserService 获取当前用户信息
  ↓
标准化为 UserProfile
  ↓
保存或更新 users 表
  ↓
返回用户信息给前端
```



```ts

  message: '二维码已过期，请刷新后重新扫码',

}
```
